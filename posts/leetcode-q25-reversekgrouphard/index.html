Problem Description Link to heading

Given the head of a linked list, reverse the nodes of the list k at a time, and return the modified list.

k is a positive integer and is less than or equal to the length of the linked list. If the number of nodes is not a multiple of k then left-out nodes, in the end, should remain as it is.

You may not alter the values in the list’s nodes, only nodes themselves may be changed.

这道题比较简单，但题干强调 “You may not alter the values in the list’s nodes, only nodes themselves may be changed.” 因此，它是 LeetCode 中一道非常经典的链表操作题。对于链表顺序的调整，只需要牢记一点——完成操作后，必须仍可以通过手中的节点访问链表中的每个节点。我们可以根据下面的例子来理解一下这句话：

此时，借助 Node(head)，我们可以访问链表 Node(1)$\rightarrow$Node(2)$\rightarrow$Node(3)$\rightarrow$Node(4)$\rightarrow$nullptr 中的任意节点。此时我们对链表的顺序进行调整，在不添加新节点的基础上将链表 Node(1)$\rightarrow$Node(2)$\rightarrow$Node(3)$\rightarrow$Node(4)$\rightarrow$nullptr 更新为 Node(2)$\rightarrow$Node(1)$\rightarrow$Node(3)$\rightarrow$Node(4)$\rightarrow$nullptr。如果我们执行命令

此时，基于已知的变量 Node(head) 和 Node(cur)，我们完全无法访问 Node(3), Node(4)，换句话说，后面的链表节点已经丢失了。由于 Node(3) 的存储地址未知，我们无法通过 head->next=Node(3) 来完成调整顺序的需求。事实上，正确的命令应当是（我们将这一操作定义为反转算子）

cur->next = head;

综上所述，假设被变量直接标识的节点为“已知节点”，通过属性 next 访问单向链表后续节点的操作为“联系”。当我们操作链表时，一定要保证已知节点对未知节点的联系。基于这一思想，我们来看题干。一个非常直观的想法是，建立一个 vector<ListNode*> 容器，将 reverseKGroup 操作涉及到的节点全部存起来，这样就不用担心节点的丢失问题。但已知 1 <= k <= n <= 5000，在最坏的情况下我们要存 5000 个节点，这毫无疑问会产生巨大的存储开销，因此，我们要尽可能少地使用已知节点完成操作，具体思路可以通过一个例子来暴力分析一下。

vector<ListNode*> reverseK(ListNode* head, int k) {
    vector<ListNode*> ans = {head, head};
    if (k == 1) return ans;
    else if (k == 2) {
        ListNode *cur = head->next;
        if (cur->next == nullptr) head->next = nullptr;
        else head->next = cur->next;
        cur->next = head;
        ans = {cur, head};
        return ans;
    } else {
        ListNode *left = head, *cur = head;
        int step = 0, stateCount = 0;
        while (step <= k-2) {
            // update states--Node(head), Node(left), Node(cur)
            head = cur;
            left = head;
            cur = head->next;
            stateCount = step;
            while (stateCount > 0) {
                left = left->next;
                cur = cur->next;
                stateCount--;
            }

            // reverse operation
            left->next = cur->next;
            cur->next = head;
            step++;
        }
        // Get Node(head)
        head = cur;
        // Get Node(tail)
        ListNode *tail = head->next;
        int countTail = k - 2;
        while(countTail--) tail = tail->next;
        ans = {head, tail};
    }

    return ans;
}

reverseK() 函数，输入是子链表的 head 节点和需要反转的节点数量——k，再输出反转子链表的 head 节点和 tail 节点。显然，函数并不会影响 tail 节点和后续节点的联系，但是我们需要完成反转子链表和之前链表的联系，此时就需要我们保留前一个子链表的 tail 节点。现在我们回到题干信息，注意到 “If the number of nodes is not a multiple of k then left-out nodes, in the end, should remain as it is.” 因此，我们还需要一个函数来判断后续 k 个节点是否需要反转：

bool isNeedReverse(ListNode* head, int k) {
    int count = 1;
    if (head == nullptr) return false;
    while (count < k) {
        if (head->next == nullptr) return false;
        else head = head->next;
        count++;
    }
    return true;
}

class Solution {
public:
    bool isNeedReverse(ListNode* head, int k) {
        int count = 1;
        if (head == nullptr) return false;
        while (count < k) {
            if (head->next == nullptr) return false;
            else head = head->next;
            count++;
        }
        return true;
    }

    vector<ListNode*> reverseK(ListNode* head, int k) {
        vector<ListNode*> ans = {head, head};
        if (k == 1) return ans;
        else if (k == 2) {
            ListNode *cur = head->next;
            if (cur->next == nullptr) head->next = nullptr;
            else head->next = cur->next;
            cur->next = head;
            ans = {cur, head};
            return ans;
        } else {
            ListNode *left = head, *cur = head;
            int step = 0, stateCount = 0;
            while (step <= k-2) {
                // update states--Node(head), Node(left), Node(cur)
                head = cur;
                left = head;
                cur = head->next;
                stateCount = step;
                while (stateCount > 0) {
                    left = left->next;
                    cur = cur->next;
                    stateCount--;
                }

                // reverse operation
                left->next = cur->next;
                cur->next = head;
                step++;
            }
            // Get Node(head)
            head = cur;
            // Get Node(tail)
            ListNode *tail = head->next;
            int countTail = k - 2;
            while(countTail--) tail = tail->next;
            ans = {head, tail};
        }

        return ans;
    }

    ListNode* reverseKGroup(ListNode* head, int k) {
        if (k == 1) return head;

        vector<ListNode*> headAndTail = reverseK(head, k);
        ListNode *ans = headAndTail[0], *tail = headAndTail[1];

        while (isNeedReverse(tail->next, k)) {
            headAndTail = reverseK(tail->next, k);
            tail->next = headAndTail[0];
            tail = headAndTail[1];
        }

        return ans;
    }
};
